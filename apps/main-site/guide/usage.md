# 使い方

## 開発環境のセットアップ

### 前提条件

- Node.js 16.0.0以上
- pnpm 8.0.0以上

### インストール

リポジトリをクローンした後、以下のコマンドを実行して依存関係をインストールします。

```bash
pnpm install
```

### 開発サーバーの起動

メインサイトの開発サーバーを起動するには、以下のコマンドを実行します。

```bash
pnpm dev
```

特定のプロジェクトの開発サーバーを起動するには、以下のコマンドを実行します。

```bash
pnpm --filter project1 dev
```

### ビルド

すべてのプロジェクトをビルドするには、以下のコマンドを実行します。

```bash
pnpm build
```

特定のプロジェクトをビルドするには、以下のコマンドを実行します。

```bash
pnpm --filter project1 build
```

## ドキュメントの編集

### Markdownファイルの編集

各プロジェクトのドキュメントは、Markdownファイルで記述されています。以下のディレクトリ構造に従って編集します。

```
packages/project1/
├── en/
│   ├── v1/
│   │   ├── index.md
│   │   └── ...
│   └── v2/
│       ├── index.md
│       └── ...
└── ja/
    ├── v1/
    │   ├── index.md
    │   └── ...
    └── v2/
        ├── index.md
        └── ...
```

### フロントマターの設定

各Markdownファイルの先頭には、フロントマターを記述することができます。

```yaml
---
title: ページタイトル
description: ページの説明
---
```

### コンポーネントの使用

共通コンポーネントを使用するには、Markdownファイル内で以下のように記述します。

```md
<VersionSelector />

# ページタイトル

ページの内容...
```

## 多言語対応

### 言語の切り替え

言語を切り替えるには、URLのパスを変更します。

- 日本語: `/`
- 英語: `/en/`

### 翻訳の追加

新しい言語を追加するには、以下の手順に従います。

1. `.vitepress/config.js`の`locales`に新しい言語を追加します。
2. 各プロジェクトに新しい言語のディレクトリを作成します。

## バージョン管理

### バージョンの切り替え

バージョンを切り替えるには、`<VersionSelector />`コンポーネントを使用します。

### 新しいバージョンの追加

新しいバージョンを追加するには、以下の手順に従います。

1. 各言語ディレクトリに新しいバージョンのディレクトリを作成します。
2. `.vitepress/config.js`の`versions`に新しいバージョンを追加します。
